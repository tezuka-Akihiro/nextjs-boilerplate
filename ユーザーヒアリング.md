# 開発テンプレート：ユーザーヒアリング・設計ガイド

## 📋 このドキュメントの目的

このリポジトリを使った新プロジェクト開発のためのステップ・バイ・ステップガイドです。
AIアシスタントがこのガイドを読み、ユーザーとの対話を通じて設計完了まで進められるように設計されています。

**重要**: 現在は設計フェーズのみ対象。実装は含まない。

---

## 🚀 開発フロー概要

```
Step 1: 企画・要件定義 → Step 2: プロジェクト初期設定 → Step 3: 設計完了
     ↓                      ↓                        ↓
  企画書作成             project-overview.md         全設計書完成
                        リポジトリ設定               AI設計ガイド適用
```

## 📝 Step 1: 企画・要件定義

### 1-1. 企画書.md 作成
**目的**: プロジェクトの基本コンセプトと方向性を定義

**ユーザーに確認する項目**:
```
□ アプリケーション名（仮でもOK）
□ ターゲットユーザー（年齢・属性・ニーズ）
□ 解決したい課題・問題
□ 提供価値・差別化要因
□ 参考にしたいアプリ・サービス
□ 収益モデル（将来的な方向性）
□ 開発予算・期間の大枠
```

**作成ファイル**: `docs/planning/企画書.md`

**質問例**:
- 「どんなユーザーの、どんな課題を解決したいですか？」
- 「類似サービスと比べて、どこが差別化ポイントですか？」
- 「最初のユーザー100人は、どこから集めますか？」

### 1-2. 技術要件の大枠確認
**ユーザーに確認する項目**:
```
□ 対象プラットフォーム（Web / モバイル / 両方）
□ 想定ユーザー数（初期・1年後目標）
□ 必須の外部サービス連携（SNSログイン、決済など）
□ 特別な技術要件（リアルタイム通信、画像処理など）
□ 既存システムとの連携要件
```

## 🔧 Step 2: プロジェクト初期設定

### 2-1. リポジトリのプロジェクト登録
**手順**:
1. このボイラープレートをクローン
2. プロジェクト名でリポジトリ作成
3. `git clone --bare` でミラー作成
4. 新しいリポジトリにプッシュ

**コマンド例**:
```bash
# 1. ボイラープレートクローン
git clone https://github.com/your-org/nextjs-boilerplate.git my-new-project

# 2. 新しいリポジトリに切り替え
cd my-new-project
git remote set-url origin https://github.com/your-org/my-new-project.git

# 3. 初期プッシュ
git push -u origin main
```

### 2-2. project-overview.md の項目埋め
**目的**: 技術的な基盤情報と開発方針を確定

**ユーザーに確認・決定する項目**:
```
□ 正式なアプリケーション名
□ 具体的なターゲットユーザー定義
□ 技術スタック確認・調整
□ 主要機能の優先順位
□ セキュリティ・パフォーマンス要件
□ 外部サービス選定
```

**質問例**:
- 「ユーザー認証は、メール・パスワードとSNSログイン、どちらを優先しますか？」
- 「データはリアルタイム同期が必要ですか？」
- 「画像アップロード機能は必要ですか？容量制限はどの程度？」

**作成・更新ファイル**: `docs/project-overview.md`

## 💡 AIアシスタント向け：対話のコツ

### ヒアリングの進め方
1. **大枠から詳細へ**: 企画→機能→技術仕様の順で深堀り
2. **具体例で確認**: 「例えば〜のような場合は？」で理解度チェック
3. **選択肢の提供**: 技術的な判断が必要な場合は2-3案提示
4. **一気に決めない**: 複雑な要件は段階的に詳細化

### 注意事項
- 実装コードは書かない（設計のみ）
- CLAUDE.mdの制約（AI滑走路4層等）は説明する
- ユーザーの技術レベルに合わせて説明の詳しさを調整
- 不明な点は遠慮なく質問する
